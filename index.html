The more you knowGamePad Support

This site is built with support for the GamePad API! Everything is usable with just the gamepad, borrowing some ideas from how mouse support was built for the touch-centric iPad, with an adaptable cursor that stretches and wraps around interactable elements.

If you have a gamepad connected your your computer, just press any button to get started!

You can use the Left Stick to move the cursor, and the Right Stick to scroll the page. Use the Action Button to click! If the cursor hits the edge, the page will also scroll. Why not try it out over the gamepad illustration on the left?

The more you knowA Modern Web

This site is built with the latest web technologies, like the SVG animations you can see to the side - they are pure SVGs with CSS animations and transitions. CSS variables control the changing color, and our scripts have been written in the latest and greatest before being transpiled and bundled with ParcelJS.

The true magic of the web is working as closely to the native API as possible - no need for extra layers that wrap function calls in other function calls. You can have a look at the simplicity that forms the basis of this site on GitHub, where the source code for this site is publicly available.

For the best experience, you should be on the latest version of your browser. If your browser is no longer supported or hasn't seen a major version in two years, you are probably on Internet Explorer. Don't do that. The web has moved on.

The more you knowLighthouse Shows the Way

This site is built. And then I tested it. And then I ran Lighthouse - and it's a pretty good result if you ask me. Accessibility is high, and that doesn't even include the hidden accssibility menu and mode (just start tabbing from the top of the page to enable it permanently to receive high-contrast and a hidden menu).

Performance wise, the site is fast - time expressed in milliseconds - the longest loading is for the full .glb file that is required for the header to do its thing. 3D isn't cheap, and it requires a lot of data, from bones to armatures and skin weights to vertices and faces, normals and more.

Note: I did the testing on my laptop - but I know that's not a benchmark for the general public. So I also tested it on a Surface GO with Windows, which is kind-of a little bit 💩*. But it still worked well!

* It has great battery management these days though. I leave it untouched for months, and it just starts up with most of the battery intact. Hats off where hats needs to be... off-ed?
